### Hibernate envers
Hibernate envers - модуль для автоматизации контроля версий сущностей
(Envers - Entity Versioning. Цель: Аудит данных).

В библиотеке есть концепция ревизий. 
По сути, одна транзакция представляет собой одну ревизию.
Поскольку ревизии являются глобальными и имеют номер ревизии, 
вы можете запрашивать различные объекты в этой ревизии, 
получая (частичное) представление базы данных в этой ревизии.

### Стратегии аудита
* Стандартная - при изменении в историю сохраняется новая версия 
* Аудит достоверности - при изменении в историю сохраняется и новая и старая версия

### Аннотации
* @Audited - аудит всего класса или конкретный полей
* @NoAudited - исключить из аудита поле

### Конфигурация
prefix: spring.jpa.properties.org.hibernate.envers
* audit_strategy
* audit_table_prefix
* audit_table_suffix
* store_data_at_delete
* default_schema
* default_catalog

### REV_TYPE
* 0 - создание
* 1 - редактирование
* 2 - удаление
